# Model: Platform vs Evolution Systems

## The Platform Model

Platforms (Google, Apple, Amazon, WeChat) represent a partial opening:

- **Open participation** — anyone can build on the platform
- **Centralized governance** — the platform owner controls the rules
- **Asymmetric value capture** — the platform captures a share of all value created
- **Managed evolution** — the platform owner decides what changes are allowed

Platforms are an improvement over closed industrial systems — they enable more participation.

But they are not AAA. They are:

> **Centralized systems with open interfaces.**

## The Evolution Model

An evolution system is different:

- **Open participation** — anyone can enter
- **Distributed governance** — no single entity controls the rules
- **Symmetric value capture** — value flows to where it is created
- **Unmanaged evolution** — the system evolves whether the original creator likes it or not

Examples that approach this model:
- TCP/IP (no owner, evolves through RFC process)
- Git (distributed, anyone can fork)
- Bitcoin (no central authority)
- Scientific publishing (anyone can contribute, peer review selects)

## The Key Difference

| Platform | Evolution System |
|---|---|
| "Build on our terms" | "Build on any terms" |
| We capture value | Value finds its own path |
| We decide what ships | What works, ships |
| Our APIs, our rules | Protocols, not platforms |

## Why This Matters

Platforms tend toward **extraction** over time — the platform owner's incentive is to capture more value.

Evolution systems tend toward **generation** — new value is continuously created by new participants.

AAA is a model for evolution systems, not platforms.
