# COS-SOP

## How this was made

1. **触发事件**: Fable 5 发布，媒体将其描述为"Claude Code 的竞争对手"
2. **认知重构**: 识别出媒体叙事中的类别错误——将模型（引擎）与工具（Agent 框架）置于同一比较层级
3. **层级建模**: 构建引擎层→工具层的清晰分层，展示 Claude Code 的模型灵活性
4. **反事实推理**: 如果 Fable 5 是工具，它应该有自己的 Agent 框架、CLI、IDE 集成——但它没有

## Key Decisions

1. **选择"引擎"隐喻而非"模型"术语** — "模型"在 AI 领域有特定含义，"引擎"更直观地传达了"可替换的动力源"概念
2. **不改变 10 维度排名** — Fable 5 验证了排名而非推翻它，强行调分反而削弱了框架的稳定性

## What you tried and abandoned

- 尝试为 Fable 5 单独评分并插入排名，发现它无法被评分——评分框架是为工具设计的，Fable 5 不是工具
