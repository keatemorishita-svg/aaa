# Claude Fable 5：引擎，不是竞品

> 模型 vs 工具的层级区分——纠正一个常见的认知混淆。

---

## 核心洞察

Fable 5 是**模型**（引擎），Claude Code 是**工具**（Agent 框架）。两者不在同一比较层级上。

```
Claude Code（工具/Agent 框架层）
  ├── 可装 deepseek-v4-pro 引擎（成本最低）
  ├── 可装 claude-opus-4-8 引擎
  └── 可装 claude-fable-5 引擎 ← 新选项，天花板最高

Cursor / Windsurf / Copilot Agent（工具层）
  └── 只能用厂商给定的引擎，用户不能换
```

**Fable 5 不是 Claude Code 的替代品——它是 Claude Code 的升级选项。**

---

## 对 10 维度排名的验证

- Claude Code 的排名不变（仍然是第一名 91 分）
- Fable 5 让 CC 在"意图理解深度"和"跨文件执行"维度上的天花板更高
- "模型灵活性"（维度 7，CC 满分 10 分）恰好被 Fable 5 证明——行业最强模型发布的第二天，CC 用户就能接入
- 竞争工具的分数不变，但实质差距拉大——因为它们模型锁定，接不到 Fable 5

---

## 市场启示

新模型的发布不改变工具排名，但**拉大了模型灵活的工具与模型锁定的工具之间的实质差距**。

讨论新模型发布时，不应问"XX 新模型是不是超过了 Claude Code"，
而应问"这个新模型能不能接入 Claude Code，从而让它更强"。
