# AAA 核心问题集

> 10 个对抗性问题——前 5 个攻击框架本身，后 5 个攻击开源战略。

---

## 第一部分：框架本身的 5 个核心问题

1. **可证伪性**: 有没有一个公认的"在演化"的开放系统不遵循 Fork → Merge？Linux 内核的增量提交是否是反例？
2. **Selection 模糊**: Selection 被排除在三大原语之外，但在人造系统中它总是被内部化。它到底是不是原语？
3. **权力不可回避**: "没有中心控制"——但谁来 Merge？每一种已知的复杂系统都有 Merge 节点。
4. **操作价值缺失**: 如果有人读完了 AAA 但什么都不做，框架失败了吗？
5. **Fork 的黑暗面**: 什么情况下 Fork 不是演化而是毁灭？Bitcoin → Bitcoin Cash，OpenAI 从非盈利到盈利。

→ 完整分析：[ANALYSIS.md](../../ANALYSIS.md)

## 第二部分：开源的 5 个核心问题

1. **MIT 许可证矛盾**: MIT 允许 Fork 后永不 Merge——这在框架自身逻辑下是"寄生"而非"演化"
2. **治理虚伪风险**: GitHub 仓库天然中心化（Owner 有最终 Merge 权），如何不表演自己所批判的东西？
3. **成功不可测量**: 非代码项目的成功没有标准指标——Stars 是虚荣，Fork 可能是空的，实际使用无法追踪
4. **Fork 反噬原版**: 如果 Fork 版本比原版更成功，从框架逻辑看这是胜利，但作为项目创始人这是死亡
5. **维护者 = 新中心**: 你越认真维护项目，你越成为你批判的那种"中心"
