# COS-SOP

## How this was made

1. **需求分析**: 用户的工作范式被确认为 Agent-first（AI 写代码，人审核方向），这决定了维度必须以 Agent 自主性为中心
2. **维度设计**: 从实际使用体验中提取 10 个可评分的维度，覆盖从理解到执行到成本的全链路
3. **数据收集**: 对 7 款工具的每个维度进行 1-10 评分，基于实际使用而非文档声明
4. **排名验证**: 积分排序后，金字塔可视化确认结构合理性
5. **洞察提取**: 识别出"范式差距（19 分）> 技术差距"的核心结论

## Key Decisions

1. **Agent-first 而非补全-first** — 传统工具对比以补全速度/准确率为核心，本框架以 Agent 自主执行能力为中心。理由：用户的范式决定了什么指标重要。
2. **模型灵活性作为独立维度** — 大多数工具对比忽略这一点，但在 API 可接入时代，这是护城河级别的差异。
3. **10 维度而非 3-5 维度** — 太少会丢失区分度（Claude Code 和 Copilot Agent 在 5 维下差距不明显），太多会增加评分噪音。

## What you tried and abandoned

- 尝试加入"社区生态"维度（插件数、社区活跃度），但发现这个维度与"工具实际能力"相关性弱，且变化太快不适合静态排名
- 尝试机器自动评分（跑 benchmark），但 benchmark 分数无法反映"与人的协作体验"——而这恰恰是 Agent-first 范式下的核心需求
