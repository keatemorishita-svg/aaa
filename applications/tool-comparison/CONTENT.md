# AI 编程工具 10 维度对比分析

> Agent-first 视角下的全景对比、积分排名与金字塔结构

---

## 背景

以"Agent 自主执行能力"为中心，从 10 个维度对比 7 款 AI 编程工具。用户范式：Agent-first（AI 写代码，人定方向和审核）。

---

## 10 维度定义

1. **意图理解深度** — 从模糊需求到可执行计划的能力
2. **跨文件规划与执行** — 自主决定改哪些文件、按什么顺序
3. **错误自愈能力** — 生成代码→运行→报错→自己修，不贴 stack trace
4. **上下文持久性** — 长会话中不"失忆"，保持逻辑一致
5. **输出质量与审核友好度** — diff 干净，代码风格一致，审核不累
6. **成本效率** — 完成一个完整任务的 token/金钱总消耗
7. **模型灵活性** — 能否自带 API Key、切换底层模型
8. **工具调用广度** — 终端命令、读日志、搜索、git、测试的自主半径
9. **自定义与记忆** — CLAUDE.md、rules、memory 的可定制性
10. **视觉与交互体验** — UI/Diff 可视化、审核效率

## 积分总表

| 排名 | 工具 | 总积分 | 一句话 |
|:---:|------|:---:|------|
| 1 | Claude Code | 91 | 最强大脑，支持 DeepSeek API 是决定性优势 |
| 2 | Cursor Agent | 72 | UI 最好的 Agent，但模型锁定是硬伤 |
| 3 | Aider | 68 | 开源版 CC 替代品，成本同样极低 |
| 4 | Windsurf Flow | 67 | 最自动、最省心，但也最不精确 |
| 5 | Cody | 62 | 大型代码库的 X 光机（Sourcegraph 底座），但 Agent 弱 |
| 6 | Copilot Agent | 57 | 生态最强，Agent 刚起步 |
| 7 | Amazon Q | 51 | AWS 原生场景不可替代，通用场景乏力 |

---

## 核心洞察

Claude Code 的 91 分和 Cursor 的 72 分之间的 19 分差距不是技术差距，是范式差距。
Claude Code 设计时就选择了 Agent-first，而 Cursor 是从补全-first 演进到 Agent 的。
后者有历史包袱，前者轻装上阵。
