# AXA v3 文件命名规范

> 一个最小协议的设计实例：信息密度、排序、可读性。

---

## 格式

```
{倒序分}{G}{A}__{日期}__{标题}.md
```

---

## 标记含义

| 标记 | 含义 | 谁来加 | 示例 |
|------|------|--------|------|
| 倒序分 | `99 - asset_score`，高分排前面 | 代码自动 | `14` = 85分 |
| **G** | 论文有 GitHub 开源仓库 | 代码自动 | `17G__...` |
| **A** | 用户已手动解构过这篇论文 | 用户手动加 | `17GA__...` |

---

## 排序规则

Obsidian 按文件名字符串升序排列：
- 分数越高 → 倒序分越小 → 排越前面
- `14GA__` (85分, 有GitHub, 已解构) 排最前
- `59__` (40分, 无GitHub, 未解构) 排最后

---

## 设计原则

1. **信息密度** — 文件名本身携带三维元数据（分数、仓库、解构状态），无需打开文件即可判断优先级
2. **排序即过滤** — 利用文件管理器的原生排序实现优先级排列，零额外工具
3. **渐进标注** — G（自动）→ A（手动），先自动后手动，不增加初始成本
4. **最小协议** — 只有 2 个标记位 + 1 个分数位，够用但不膨胀
