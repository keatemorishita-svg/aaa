# COS-EVOLUTION

## Current State
- **Maturity**: [x] Stable — v3 已被日常使用验证
- **Known weaknesses**: 仅适配 Obsidian 的排序规则；标记位数量有天然上限（文件名长度限制）

## Evolution Invitation
- **Best contribution type**: Fork — 适配你的文件管理器和你的元数据维度
- **What needs improvement most**: 扩展到更多文件管理器（Finder、Explorer、Zotero）
- **What should NOT be changed**: 倒序分逻辑（这是协议的核心创新）

## Fork Policy
- **Fork friendly?**: [x] Yes, fork anything — 命名协议是最适合 Fork 的应用类型
- **Merge preference**: 适配新平台/工具的版本欢迎 Merge
- **Succession**: 协议的生命周期取决于使用者的实际需求——如果 Obsidian 改变了排序规则，协议需要相应进化
