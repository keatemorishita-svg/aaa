# The Evolution Loop

## Fork → Experiment → Selection → Merge → Repeat

This is not a software pattern. It is a universal mechanism.

---

## The Four Stages

### 1. Fork (Divergence)

A copy of the system is made at a point in time.

This creates a **branch** — an independent line of evolution.

Forking is the mechanism of **exploration**. Without it, the system can only walk a single path.

Examples:
- Genetic mutation creates a variant organism
- GitHub fork creates an independent codebase
- A new scientific hypothesis diverges from orthodoxy
- A startup challenges an incumbent's business model

### 2. Experiment (Variation)

The fork undergoes changes in its local environment.

This is where **mutation** happens — changes to structure, behavior, or configuration.

Examples:
- Natural selection tests the variant organism
- A forked repo adds features or fixes bugs
- A hypothesis is tested against evidence
- A startup iterates on product-market fit

### 3. Selection (Filtering)

The environment selects which variants survive.

This is the most complex and varied stage. Selection mechanisms include:
- **Reputational** — peer review, community voting
- **Market** — user adoption, revenue
- **Computational** — automated testing, benchmarks
- **Biological** — survival and reproduction

### 4. Merge (Integration)

Surviving variants are reintegrated into the main line of evolution.

This requires **compatibility** — the variant must be able to rejoin the system.

Examples:
- Pull request merged into main
- Accepted paper becomes part of the scientific canon
- Successful startup becomes industry standard
- Advantageous gene spreads through population

---

## Why a loop?

Each stage feeds the next. The output of Merge becomes the starting point for the next Fork.

A system that completes this loop continuously is **alive** in an evolutionary sense.

A system that breaks any stage of the loop is **frozen**.

---

## Properties of the Loop

- **Non-teleological** — there is no final destination, only continuous adaptation
- **Distributed** — no single entity controls all stages
- **Recursive** — the loop itself can evolve (meta-evolution)
- **Fractal** — the same loop operates at every scale, from genes to civilizations
