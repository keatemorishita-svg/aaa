# Kernel — Core Definition

## What is AAA?

AAA is a **minimal generative grammar** for open systems.

It defines the smallest set of conditions under which a system can continuously evolve without centralized control.

---

## The Kernel

The kernel consists of exactly two components:

### 1. The Evolution Loop

```
Fork → Experiment → Selection → Merge → Repeat
```

This is the universal mechanism by which open systems adapt, improve, and persist.

### 2. The Three Primitives

Every open system must satisfy:

- **Access** — the system can be entered by new participants
- **Mutation** — the system can be modified by those participants
- **Recursion** — the system can accept and integrate its own modifications

---

## Why a kernel?

A kernel is the smallest possible definition of a system.

The thinner the kernel, the harder it is to refute.
The thinner the kernel, the wider the range of systems it describes.

AAA does not specify:
- What selection mechanism to use
- What protocols to implement
- What technology to build on
- Who participates

It only specifies **the conditions under which evolution is possible**.

---

## Invariants

These are not open to change within AAA:

1. The Evolution Loop is the universal mechanism
2. The Three Primitives are necessary conditions for open evolution
3. The system must be open to its own redefinition

Everything else — architecture, models, examples — is interpretation.
