# Architecture — Protocol Layer

## Function: Interaction Rules

The Protocol Layer defines how components of the system interact with each other.

---

## Core Questions

- How do participants communicate?
- What are the rules of engagement?
- How are conflicts resolved?
- How do new participants learn the rules?

---

## Components

### Communication Protocols

Standards for information exchange:
- Languages (natural and programming)
- Data formats (APIs, schemas, standards)
- Signaling mechanisms (broadcast, point-to-point, publish-subscribe)

### Governance Protocols

Rules for decision-making:
- Voting mechanisms
- Consensus processes
- Delegation chains
- Dispute resolution

### Coordination Protocols

Mechanisms for collective action:
- Markets (price signals)
- Organizations (hierarchies, teams)
- Networks (peer-to-peer, federated)

---

## In AAA

Protocols are **not** the same as laws.

Laws are fixed rules enforced by a central authority.

Protocols are **evolving conventions** that persist because they are useful.

A healthy protocol layer allows protocols to:
- Be proposed
- Be adopted
- Be challenged
- Be replaced

---

## Properties of a Healthy Protocol Layer

- **Documented** — rules are explicit and accessible
- **Evolvable** — rules can change through defined processes
- **Compatible** — new protocols can interoperate with existing ones
- **Minimal** — only necessary constraints are imposed
